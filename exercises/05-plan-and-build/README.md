# Exercise 05 — Plan & Build 🏗️

**Goal:** Use Copilot to plan and build something from scratch.

---

## What you'll learn

By the end of this exercise, you'll know how to:
- Use `/plan` to let Copilot think before it acts
- Build a simple project using only plain English
- Review and approve Copilot's plan before it runs

---

## Why use a plan?

For small tasks, you can just ask Copilot directly.
But for bigger tasks — like creating a new file or building something with multiple parts —
it's better to ask Copilot to **make a plan first**.

This way, you can see exactly what Copilot intends to do before it does anything.

---

## The Challenge

You're going to ask Copilot to build you a **personal journal template** — a file you could
use to write your thoughts each day.

---

## Step 1 — Switch to Plan mode

Press `Shift+Tab` to switch into **Plan mode**. You'll see the label change.

---

## Step 2 — Describe what you want

Type something like:
```
Create a simple daily journal template file called journal.txt.
It should have space for the date, how I'm feeling, three things I'm grateful for,
and one goal for the day.
```

Copilot will create a **plan** and show it to you before doing anything.

---

## Step 3 — Review the plan

Read through what Copilot plans to do.

- If it looks good, approve it!
- If you want to change something, type your feedback and Copilot will update the plan.

---

## Step 4 — Let Copilot build it

Once you approve, Copilot will create the file.

After it's done, type:
```
@journal.txt show me what you created
```

---

## Step 5 — Customize it

Now ask Copilot to adjust it to your liking:
- `@journal.txt can you add a section for "what I learned today"?`
- `@journal.txt make the sections easier to fill in with blank lines`
- `@journal.txt add an inspirational quote at the top`

---

## ✅ Congratulations — you've finished all 5 exercises! 🎉

You now know how to:
- ✅ Ask Copilot anything in plain English
- ✅ Use `@` to share files with Copilot
- ✅ Ask Copilot to edit and create files
- ✅ Use slash commands
- ✅ Use `/plan` to build things step by step
