# Exercise 02 — Read & Explain 🔍

**Goal:** Learn how to ask Copilot to read and explain a file to you.

---

## What you'll learn

By the end of this exercise, you'll know how to:
- Share a file with Copilot using `@`
- Ask Copilot to explain what code does
- Ask Copilot to translate "code language" into plain English

---

## Background

In this folder there's a file called `mystery.js`.

It's written in a programming language called JavaScript. Don't worry —
you don't need to understand it! That's what Copilot is for.

---

## Step 1 — Look at the file first

In Copilot CLI, type:
```
@mystery.js show me what's in this file
```

You'll see the code. It might look confusing — that's completely normal!

---

## Step 2 — Ask Copilot to explain it

Type:
```
@mystery.js can you explain what this code does in plain English?
```

Then try:
```
@mystery.js explain this like I'm 10 years old
```

---

## Step 3 — Ask detailed questions

Pick any part of the explanation you're curious about and ask more:

- `What does "function" mean in code?`
- `What is a variable?`
- `Why would someone write code like this?`

---

## Step 4 — Ask Copilot to improve it

Type:
```
@mystery.js can you add a comment to each line explaining what it does?
```

A "comment" is a note written inside code — Copilot can add them in plain English!

---

## ✅ You're done!

You just learned how to use Copilot as a personal code translator.
Move on to **[Exercise 03 — Edit Files](../03-edit-files/README.md)** when you're ready!

---

> **Tip:** Copilot can explain ANY file to you — code, config files, anything. Just use `@`.
